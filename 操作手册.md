# 第一部分：基础知识 —— 认识你电脑里的"零件"

在开始之前，你需要知道你的电脑里有哪些"工人"在帮忙干活。

## 1.1 什么是 GPU（显卡）？

想象你的电脑是一个工厂：

- **CPU**（中央处理器） = 工厂的总经理。什么事都能干，但一次只能干一件事。
- **GPU**（显卡 / 图形处理器） = 工厂里的 10000 个流水线工人。每个工人只会做简单的计算，但 10000 个人同时干活，速度极快。

AI 模型需要做海量的简单计算，所以必须用 GPU。**本项目需要一块 NVIDIA 显卡**（俗称"N 卡"），显存至少 8GB。

> **你的显卡：NVIDIA GeForce RTX 4080，显存 16GB。**  
> 这相当于有 16000 个小工人在同时干活，非常充裕。

## 1.2 什么是显存（VRAM）？

显存 = GPU 专用的"草稿纸"。

GPU 计算时需要把数据临时写在草稿纸上。显存越大，草稿纸越大，能同时放下的数据就越多。

| 比喻 | 实际含义 |
|---|---|
| 草稿纸大小 | 显存容量（GB） |
| 草稿纸上写的字 | 模型参数（AI 学到的"知识"） |
| 用完擦掉 | 释放显存 |

**本项目占用约 5.6 GB 显存**（相当于在 16GB 的草稿纸上用了约 1/3）。

## 1.3 什么是 Conda（环境管理器）？

Python 项目需要安装很多"工具包"（官方叫"依赖"或"库"）。不同项目需要的工具包版本可能不同。如果都装在同一个地方，会打架。

Conda 就是给每个项目建一个"独立房间"：
- 房间 A 里装工具包版本 1.0
- 房间 B 里装工具包版本 2.0
- 互相不影响

```
H:\anaconda3\envs\webnovel-embed\   ← 这就是我们项目的"独立房间"
```

**你只需要知道**：在使用本项目的任何命令之前，确保你在正确的房间里。启动脚本 `start_local_server.bat` 已经帮你自动做了这件事。

## 1.4 什么是命令行（终端 / PowerShell / CMD）？

命令行就是一个"文字界面"，你打字告诉电脑做什么，电脑回文字告诉你结果。

打开方式：
- 按 `Win + R`，输入 `powershell`，回车
- 或者在文件夹空白处 `Shift + 右键` → "在此处打开 PowerShell 窗口"

> **本手册中所有 `代码块` 都是要输入到命令行里的命令。**  
> 命令行里粘贴的快捷键是 `Ctrl + V`（不是右键粘贴）。

## 1.5 什么是端口（Port）？

电脑上运行着很多程序。为了让程序之间不搞混，每个网络服务分配一个"门牌号"，这就是端口。

```
http://localhost:9997
       │         └── 端口号（门牌号）
       └── localhost = 你自己的电脑
```

**本项目的服务在 9997 号端口等待请求**。

# 第二部分：项目全貌 —— 这些文件是干什么的

你的项目文件夹在 `H:\Code\mynovel\`。下面逐一解释每个文件和文件夹。

## 2.1 文件清单

```
H:\Code\mynovel\
│
├── .env                          配置文件（最重要！）
├── start_local_server.bat        启动按钮（双击即可）
├── stop_local_server.bat         停止按钮（释放显卡）
├── INSTALL_GUIDE.md              英文安装手册
├── 操作手册.md                    你正在看的这个文件
├── requirements.txt              需要安装的 Python 工具包清单
│
├── .hf_cache\                    模型缓存（AI 的大脑数据，约 9 GB）
│   └── hub\
│       ├── models--Qwen--Qwen3-Embedding-4B\   ← Embedding 模型
│       └── models--jinaai--jina-reranker-v3\   ← Reranker 模型
│
├── .opencode\                    Webnovel Writer 核心文件
│   ├── scripts\
│   │   ├── local_embed_serve.py  推理服务主程序（启动它 → AI 跑起来）
│   │   └── data_modules\
│   │       ├── api_client.py     负责与 AI 服务通信
│   │       └── config.py         读取 .env 配置
│   └── skills\                   各种写作 skill 的定义
│
└── .git\                         Git 版本管理（不要动）
```

## 2.2 逐文件详解

### `.env` —— 配置文件

这是**最重要的文件**。它告诉项目："AI 服务在哪里？叫什么名字？"

```env
# Embedding 服务（把文字变成数字）
EMBED_BASE_URL=http://localhost:9997    ← 服务地址
EMBED_MODEL=qwen3-embedding-4b          ← 使用的模型名字
EMBED_API_KEY=not-needed                ← 本地服务不需要密码

# Rerank 服务（给搜索结果排序）
RERANK_BASE_URL=http://localhost:9997   ← 同上
RERANK_MODEL=jina-reranker-v3           ← 使用的模型名字
RERANK_API_KEY=not-needed               ← 本地服务不需要密码
```

> **通俗解释**：这就像在手机里存了一个联系人的电话号码。项目需要 AI 服务时，先看这个文件找到"电话号码"（地址），然后打电话过去。

### `start_local_server.bat` —— 启动按钮

双击这个文件 = 启动 AI 推理服务。

里面只有 4 行内容：

```bat
@echo off                                          ← 不打印多余信息
set HF_HOME=H:\Code\mynovel\.hf_cache              ← 告诉程序：模型文件在 H 盘
set HF_ENDPOINT=https://hf-mirror.com              ← 告诉程序：从镜像站下载（国内快）
H:\anaconda3\envs\webnovel-embed\python.exe ...    ← 用我们的 Python 运行服务
```

> **通俗解释**：就像汽车的启动按钮。按一下，引擎就转起来了。

### `stop_local_server.bat` —— 停止按钮

双击这个文件 = 关闭 AI 服务，释放显存。

**为什么需要手动停止？** AI 服务启动后，模型一直占用显存。不停止的话，你玩游戏、剪视频可能会卡。停止后显存就释放了。

### `.hf_cache\` —— 模型缓存

AI 模型的"大脑数据"存放在这里。总大小约 9 GB。

两个子文件夹分别对应两个模型：
- `Qwen--Qwen3-Embedding-4B`：Embedding 模型（把文字变成数字的工人），约 7.7 GB
- `jinaai--jina-reranker-v3`：Reranker 模型（给搜索结果排序的工人），约 1.2 GB

> **这些文件不要手动删除！** 删了之后下次启动服务会重新下载（很慢）。

### `local_embed_serve.py` —— 推理服务的心脏

这是整个服务的核心代码。它做了三件事：
1. **启动时**：加载两个 AI 模型到显卡
2. **收到请求时**：用模型处理数据，返回结果
3. **关闭时**：释放显卡资源

你不需要理解代码细节。只要知道：这个文件在，AI 服务就能跑。

# 第三部分：核心概念 —— AI 模型到底在干什么

## 3.1 什么是 Embedding？

**Embedding = 把一段文字变成一串数字。**

比如，句子 `"今天天气真好"` 经过 Embedding 模型处理后，变成了一串 2560 个小数：

```
[0.023, -0.015, 0.008, ..., 0.031]   ← 共 2560 个数字
```

**为什么要把文字变成数字？** 因为电脑只懂数字，不懂文字。变成数字后，电脑就能比较两段话的"意思是否相近"了。

> **比喻**：Embedding 就像一个翻译官，把人类的语言翻译成电脑能理解的"数字语言"。

**本项目使用的模型**：`Qwen3-Embedding-4B`
- 4B = 40 亿个参数（相当于工人有 40 亿条经验）
- 使用 INT8 压缩技术，显存占用从 8GB 降到 4GB
- 输出的每个向量是 2560 维（2560 个数字来描述一段文字的意思）

## 3.2 什么是 Reranker？

**Reranker = 给搜索结果重新排序，把最相关的排到最前面。**

假设你搜索 `"怎样养猫"`，初步找到了 50 篇文章。Reranker 会逐一比较每篇文章和搜索词的相关性，给每篇打一个分数，然后按分数从高到低排列。

```
搜索词: "怎样养猫"

重排序前（随便排的）:        重排序后（Reranker 处理后）:
1. 狗的饲养方法              1. 🥇 猫咪饲养完全指南  ← 最相关
2. 猫咪饲养完全指南          2. 🥈 养猫新手指南
3. 养猫新手指南              3. 🥉 宠物护理大全
4. 量子物理入门              4. 狗的饲养方法
5. 宠物护理大全              5. 量子物理入门
```

**本项目使用的模型**：`jina-reranker-v3`
- 只有 6 亿个参数（是 Embedding 模型的 1/7）
- 显存占用仅 ~1.2 GB
- 一次能处理 64 篇文章，上下文长度 13 万字符

## 3.3 为什么两个模型加起来才 5.6 GB 显存？

因为用了"压缩技术"：

| 模型 | 原始大小 | 压缩方式 | 实际显存 |
|---|---|---|---|
| Qwen3-Embedding-4B | 8 GB | **INT8** (8位整数) | ~4.4 GB |
| jina-reranker-v3 | 1.5 GB | **FP16** (半精度) | ~1.2 GB |
| **合计** | **9.5 GB** | | **~5.6 GB** |

> **通俗解释**：就像把照片从 PNG 格式（大文件）转成 JPG 格式（小文件），画质几乎不变，但体积小了很多。

## 3.4 数据流程：一次完整的请求

```
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│ 你的小说  │────→│ Embedding模型 │────→│  一串数字     │
│ 章节内容  │     │ (变成数字)    │     │  (2560个小数) │
└──────────┘     └──────────────┘     └──────┬───────┘
                                             │
                    ┌──────────────┐          │
                    │  数据库       │←─────────┘
                    │ (存储数字)    │
                    └──────┬───────┘
                           │
┌──────────┐     ┌──────────────┐     ┌──────────────┐
│ 搜索结果  │←────│  Reranker    │←────│  搜索关键词   │
│ (排序后)  │     │ (重新排序)    │     │  + 候选文章   │
└──────────┘     └──────────────┘     └──────────────┘
```

# 第四部分：日常操作 —— 每天都要做的事

## 4.1 你需要做的事（按顺序）

### 步骤 ①（已自动化，可选手动）

**大多数时候你不需要手动启动服务。** 当你运行写作命令（如 `/webnovel-write`）时，项目会**自动检测并在后台启动** AI 服务。你会看到类似这样的提示：

```
本地 AI 服务未启动，正在自动启动...
等待本地 AI 服务就绪（首次需加载模型，约 30-60 秒）...
本地 AI 服务已就绪
```

**只有在你想手动控制时**，才需要双击 `start_local_server.bat`。

### 步骤 ①备选：手动启动

```
双击 start_local_server.bat
```

**你会看到什么？** 一个黑色窗口弹出，里面滚动着文字。这是服务启动的日志。

**启动过程分三个阶段：**

```
第一阶段（约 10 秒）         第二阶段（约 15 秒）         第三阶段
┌─────────────────┐       ┌─────────────────┐       ┌─────────────────┐
│ [1/2] Loading   │  →    │ [2/2] Loading   │  →    │ Ready! VRAM used │
│ embedding: Qwen │       │ reranker: jina  │       │ 5.6GB, Uvicorn   │
│ ... (INT8)      │       │ ... (FP16)      │       │ running on 9997  │
└─────────────────┘       └─────────────────┘       └─────────────────┘
    Embedding 模型加载         Reranker 模型加载          服务启动成功！
```

**启动成功的标志**：看到以下文字：
```
INFO: Uvicorn running on http://0.0.0.0:9997 (Press CTRL+C to quit)
```

> **注意**：首次启动需要下载模型（约 9 GB），会慢一些（5-15 分钟）。之后启动只需约 30 秒（模型已缓存）。

**启动后窗口能关吗？** **不能。** 这个窗口开着 = 服务在运行。关闭窗口 = 服务停止。让它最小化待在任务栏即可。

### 步骤 ② 使用服务（正常写小说）

服务启动后，你可以进行写小说的正常操作。本项目会自动连接本地服务，你不需要做任何额外的事情。

项目通过 `.env` 文件知道服务地址是 `http://localhost:9997`，自动发送请求。

### 步骤 ③ 停止 AI 服务

写完后，双击 `stop_local_server.bat`。

**为什么要停止？**
- 释放显卡显存（5.6 GB），让显卡可以用于游戏、剪视频
- 释放系统内存（模型在 CPU 内存中也占约 8 GB）
- 省电

> **如果忘记停止，会怎样？** 不会损坏电脑，只是显卡一直处于"备战"状态，可能影响其他程序的性能。

## 4.3 生成图片（封面 / 角色）

项目支持两种图片生成方式：

### 方式一：提示词输出（默认，无需配置）


之后运行生成命令，图片会自动生成并下载到本地。

# 第五部分：验证健康 —— 确认一切正常

## 5.1 健康检查（最简单的方法）

打开浏览器，在地址栏输入：

```
http://localhost:9997/health
```

**正常时**，浏览器会显示类似这样的文字：
```json
{"status":"ok","device":"cuda","gpu":"NVIDIA GeForce RTX 4080","vram_used_gb":5.6,"vram_total_gb":17.0}
```

**关键信息解读：**
| 字段 | 含义 | 期望值 |
|---|---|---|
| `status` | 服务状态 | `"ok"` |
| `device` | 使用什么计算 | `"cuda"`（用显卡） |
| `vram_used_gb` | 显卡用了多少显存 | 约 `5.6` |
| `vram_total_gb` | 显卡总显存 | `17.0` |

## 5.2 测试 Embedding（文字→数字）

在命令行中运行：

```powershell
curl -X POST http://localhost:9997/v1/embeddings -H "Content-Type: application/json" -d '{"input":["测试句子"],"model":"qwen3-embedding-4b"}'
```

**正常时**，会返回一个包含 2560 个数字的数组。

## 5.3 测试 Reranker（排序）

在命令行中运行：

```powershell
curl -X POST http://localhost:9997/v1/rerank -H "Content-Type: application/json" -d '{"query":"如何养猫","documents":["猫咪饲养指南","量子物理教程","养猫新手指南"],"model":"jina-reranker-v3"}'
```

**正常时**，`猫咪饲养指南` 和 `养猫新手指南` 的分数应该比 `量子物理教程` 高。

## 5.4 查看显卡使用情况

在命令行中运行：

```powershell
nvidia-smi
```

你会看到一个表格。关注以下几项：

```
+-----------------------------------------------------------------------------------------+
| NVIDIA-SMI 581.42                 Driver Version: 581.42         CUDA Version: 13.0     |
|-----------------------------------------+------------------------+----------------------+
| GPU  Name                  Memory-Usage | GPU-Util  Compute M. |
|=========================================+========================+======================|
|   0  NVIDIA GeForce RTX 4080    5643MiB |    0%      Default |
+-----------------------------------------------------------------------------------------+
```

| 字段 | 含义 | 正常值 |
|---|---|---|
| `Memory-Usage` | 显存使用量 | 服务运行时约 5643 MiB (~5.6 GB) |
| `GPU-Util` | GPU 计算利用率 | 无请求时 ~0%，推理时可能几十% |

# 第六部分：故障排除 —— 出问题了怎么办

> **遇到问题时，首先做这一步：重新启动！** 关闭所有窗口，重新双击 `start_local_server.bat`。

## 6.1 启动类问题

### ❌ 问题：双击 bat 文件后窗口一闪就关了

**原因**：找不到 Python 或路径不对。

**解决**：
1. 右键 `start_local_server.bat` → 编辑
2. 检查第三行的 Python 路径是否正确：
   ```bat
   H:\anaconda3\envs\webnovel-embed\python.exe ...
   ```
3. 如果路径不对，改成正确的路径后保存，重新双击。

### ❌ 问题：窗口显示很长的一堆 Python 报错

**原因**：Python 环境有问题，或者模型文件损坏。

**解决**：
1. **注意报错信息的关键词**：
   - `OutOfMemory` / `OOM` → 显存不够（见下方 6.2）
   - `ModuleNotFoundError` → 缺少工具包（见下方 6.3）
   - `ConnectionError` / `Timeout` → 网络问题（见下方 6.4）
   - `ACCESS_VIOLATION` → 显卡驱动或内存问题（重启电脑试试）
   - `commitment limit` → 系统虚拟内存不够（见下方 6.5）

### ❌ 问题：一直卡在 "Loading weights: XX%"

**原因**：模型文件在下载或加载中。

**判断**：
- 百分比在涨（比如从 30% 到 50%）→ 正常，耐心等待
- 百分比卡住不动超过 5 分钟 → 网络问题，关掉重开

## 6.2 显存相关

### ❌ 问题：CUDA out of memory（显存不够）

**完整排查**：
1. 打开命令行，输入 `nvidia-smi`，查看还有多少空闲显存
2. 关闭其他使用显卡的程序（浏览器、游戏、视频剪辑软件）
3. 重新启动服务

**如果你的显卡显存小于 8GB**，可以：
- 编辑 `local_embed_serve.py`，把 `load_in_8bit=True` 改成 `load_in_4bit=True`（进一步压缩）

## 6.3 缺少工具包

### ❌ 问题：No module named 'xxx'

**解决**：
```powershell
# 激活环境
conda activate webnovel-embed

# 安装缺少的包（把 xxx 替换为报错中的包名）
pip install xxx
```

常见缺少的包和安装命令：

| 报错中的包名 | 安装命令 |
|---|---|
| `torch` | `pip install torch` |
| `transformers` | `pip install transformers` |
| `bitsandbytes` | `pip install bitsandbytes` |
| `fastapi` | `pip install fastapi` |
| `uvicorn` | `pip install uvicorn` |

## 6.4 网络问题

### ❌ 问题：模型下载很慢或失败

**原因**：HuggingFace 官方服务器在国外，国内访问慢。

**解决方案 1（已配置）**：启动脚本中已设置了镜像站：
```bat
set HF_ENDPOINT=https://hf-mirror.com
```

**解决方案 2（手动下载）**：
1. 浏览器打开 `https://hf-mirror.com/Qwen/Qwen3-Embedding-4B`
2. 点击 "Files and versions" 标签
3. 下载所有 `.safetensors` 和 `.json` 文件
4. 放到 `H:\Code\mynovel\.hf_cache\hub\models--Qwen--Qwen3-Embedding-4B\snapshots\xxx\` 目录下
5. 同样操作 `https://hf-mirror.com/jinaai/jina-reranker-v3`

## 6.5 系统资源问题

### ❌ 问题：电脑很卡

**原因**：AI 服务占用了显卡和内存资源。

**解决**：
- 用完后及时双击 `stop_local_server.bat`
- 不在使用时不要一直开着服务

### ❌ 问题：虚拟内存不足（commit limit 错误）

**解决**：
1. 右键"此电脑" → 属性 → 高级系统设置
2. 高级 → 性能 → 设置 → 高级 → 虚拟内存 → 更改
3. 选择一个有空间的盘（比如 H: 盘）
4. 设置初始大小 16384 MB，最大 32768 MB
5. 点击"设置"，确定，重启电脑

## 6.6 服务无响应

### ❌ 问题：浏览器打不开 http://localhost:9997/health

**排查步骤**：
1. **服务是否在运行？** 看一眼启动 bat 文件弹出的黑色窗口还在不在
2. **端口是否被占用？** 命令行输入：
   ```powershell
   netstat -ano | findstr ":9997"
   ```
   如果没输出 → 服务没在运行，重新启动
   
3. **防火墙是否拦截？** 检查 Windows 防火墙设置

## 6.7 彻底恢复

如果所有方法都试过了还是不行，执行"完全重装"：

```powershell
# 1. 删除旧环境
conda remove -n webnovel-embed --all -y

# 2. 删除旧缓存
Remove-Item H:\Code\mynovel\.hf_cache -Recurse -Force

# 3. 重新创建环境
conda create -n webnovel-embed python=3.10 -y

# 4. 重新安装
conda activate webnovel-embed
pip install torch transformers fastapi uvicorn bitsandbytes accelerate sentencepiece

# 5. 重新启动
start_local_server.bat
```

# 第七部分：维护与清理 —— 让电脑保持健康

## 7.1 磁盘空间检查

### 当前占用清单

| 位置 | 内容 | 占用空间 |
|---|---|---|
| `H:\Code\mynovel\.hf_cache\` | 模型权重 | ~9 GB |
| `H:\anaconda3\envs\webnovel-embed\` | Python 环境 | ~5.6 GB |
| **合计** | | **~14.6 GB** |

> **好消息：** 所有文件都在 H: 盘，C: 盘没有任何大型文件！

### 查看 C: 盘确认

```powershell
# 检查 C: 盘是否还有残留
Get-ChildItem "$env:USERPROFILE\.cache\huggingface" -Recurse -File -EA SilentlyContinue | Measure-Object Length -Sum
```

如果输出是 `0` 或很小（< 10 MB），说明 C: 盘是干净的。

### 清理不需要的模型

如果以后下载了新的模型、旧的不需要了：

```powershell
# 查看所有缓存的模型
Get-ChildItem H:\Code\mynovel\.hf_cache\hub -Directory | ForEach-Object { $_.Name }

# 删除某个模型（把 xxx 替换为模型文件夹名）
Remove-Item "H:\Code\mynovel\.hf_cache\hub\xxx" -Recurse -Force
```

## 7.2 显卡维护

### 监控显卡温度

```powershell
nvidia-smi --query-gpu=temperature.gpu --format=csv
```

正常温度：30°C - 80°C。如果超过 85°C，考虑清理灰尘或加强散热。

### 更新显卡驱动

定期检查 NVIDIA 官网或 GeForce Experience，保持驱动最新。

## 7.3 更新项目

当 Webnovel Writer 项目有更新时：

```powershell
cd H:\Code\mynovel
git pull
```

更新后重新启动服务即可。

# 第九部分：AI 小说共创 —— 协作模式

## 9.1 项目定位

本项目（《仙界逃兵》）是一个 **AI 辅助创作的业余小说项目**，同时也是一套可供他人复用的写作框架。

| 角色 | 职责 |
|------|------|

| 共创者 | 设定建议、情节讨论、素材提供（无经济报酬） |

## 9.2 如果你是本项目的使用者（作者本人或维护者）

按照本手册前八部分正常操作即可。

## 9.3 如果你是共创参与者

### 你不需要搭建本地环境

共创者**不需要**安装 Python、Conda、模型等环境。你只需要：

1. 阅读 [`README.md`](../README.md) 了解项目
2. 阅读 [`CONTRIBUTING.md`](../CONTRIBUTING.md) 了解贡献流程和 CLA
3. 阅读 [`RISK_DISCLOSURE.md`](../RISK_DISCLOSURE.md) 了解风险
4. 通过 **GitHub Issues / Discussions / Pull Requests** 参与讨论

### 你可以贡献什么

- **世界观设定**：势力设计、历史事件、地理版图
- **角色设定**：角色原型、性格、关系线
- **情节走向**：冲突设计、爽点构思、伏笔建议
- **现实案例素材**：社会现象（需遵循三层虚构化规范）
- **设定勘误**：一致性检查、逻辑漏洞

### 三层虚构化规范

所有现实案例必须经过以下处理后才能纳入小说：

1. **时空架空** — 将现代事件转化为修仙世界的设定
2. **身份转换** — 将现实人物的身份转换为修仙角色
3. **矛盾提炼** — 提炼核心冲突逻辑，而非复刻具体事件
