# 参与共创指南

> ⚠️ 本项目欢迎**业余小说创作爱好者**参与。
> 不推荐以小说为生的职业小说创作者参与，以免耽误其个人创作事业。

---

## 你可以参与什么

| 贡献类型 | 说明 | 方式 |
|----------|------|------|
| 世界观设定 | 势力、地理、历史、规则设计 | Issue / Discussion |
| 角色设定 | 角色原型、性格、关系线建议 | Issue / Discussion |
| 情节走向 | 冲突设计、反转创意、爽点构思 | Issue / Discussion |
| 现实案例素材 | 社会现象、人性困局（需虚构化） | Issue |
| 设定勘误 | 一致性检查、逻辑漏洞、文笔建议 | PR / Issue |
| 写作工具改进 | `.opencode/` 框架功能改进 | PR |

## 不适合贡献的

- **章节正文** — 正文由项目维护者统一创作，以保持风格一致与平台签约合规
- **未经许可的他人作品** — 包括但不限于搬运、改写他人小说内容
- **直接现实人物映射** — 未经虚构化处理的现实人物、事件描述

## 贡献流程

### 方式一：Issue 讨论（推荐入门）

1. 搜索是否已存在相关 Issue
2. 新建 Issue，描述你的想法
3. 参与讨论，等待维护者评估

### 方式二：Pull Request（设定/文档修改）

1. Fork 本仓库
2. 创建分支：`feat/你的贡献描述`
3. 修改文件（仅限 `大纲/`、`设定集/`、文档类文件）
4. 提交 PR，描述修改内容与理由

### 方式三：Discussion（长期讨论）

适合需要深度讨论的设定、情节走向等内容。

## CLA（贡献者许可协议）

**提交 Pull Request 或 Issue 建议即表示你同意以下条款：**

### 1. 版权授权

你无偿、永久、不可撤销地将本次贡献内容的**财产性著作权**（包括但不限于复制权、发行权、改编权、信息网络传播权）授权给项目维护者，用于商业出版（含但不限于番茄小说签约、实体出版、电子书发行等）。

### 2. 无经济报酬

- 本项目为**业余共创项目**，贡献者不会获得任何金钱报酬
- 番茄小说平台、或任何其他渠道产生的全部收益，归项目维护者（签约作者）所有
- 贡献者不得以任何形式主张收益分成

### 3. 署名权保留

贡献者享有署名权，署名方式为双轨制：

**A. AUTHORS.md 贡献者列表**
```
### 世界观贡献者
- @GitHubID — 提出 [具体贡献内容]
```

**B. 章节末尾标注**
```
本章共创感谢: @GitHubID（[具体贡献说明]）
```

**C. 章节开头声明**（正文在番茄发布时）
```
本章部分设定建议来自共创者，共创者不享有本章收益。
```

### 4. 原创承诺

你承诺贡献的内容为原创，不侵犯任何第三方的著作权、名誉权、隐私权等合法权益。

### 5. 拒绝 CLA

如果你不同意以上条款，**请勿提交 Pull Request**。
Issue 讨论（不涉及具体创作内容）不受此约束。

## 现实案例虚构化规范

所有基于现实的案例素材，必须经过**至少三层虚构化处理**后，方可提交：

| 层级 | 操作 | 示例 |
|------|------|------|
| 第一层：时空架空 | 将事件发生的时间和地点完全架空 | 现实职场 → 修仙宗门内务 |
| 第二层：身份转换 | 将涉及人物的身份、性别、年龄等彻底转换 | 现代老板 → 元婴期长老 |
| 第三层：矛盾提炼 | 提取核心矛盾逻辑，而非复刻具体事件经过 | 拖欠工资 → 克扣灵石配额 |

**禁止**：
- 直接描述真实人物姓名、特征、经历
- 直接引用现实事件报道原文
- 以化名方式变相映射可识别的现实人物

## AI 使用声明

本项目使用 AI 工具辅助创作：
- **人类创作**：世界观、角色、情节大纲、审核与润色
- **AI 辅助**：正文行文输出、设定一致性检查、上下文检索

贡献者也欢迎使用 AI 工具辅助构思，但在提交贡献时请说明是否使用了 AI 及使用程度。

- 不人身攻击，不政治敏感话题
- 维护者保留关闭/删除不当内容的权利
