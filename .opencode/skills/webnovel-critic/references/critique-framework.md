# 深度解剖框架

> 在审查的 Step 4 中使用。按此结构剖析每个崩点。

## 4a. 定位原文

从章节中截取 2-3 句具体原文作为证据。

对照大纲检查差距时，应用双向判断：
- 章节偏离大纲，但章节本身精彩有新意 → **改大纲**，标注"大纲需更新"
- 章节偏离大纲，且章节本身虚弱无力 → **改章节**，这才是真正的崩点
大纲是草稿，章节是成品。好章节让大纲臣服。

## 4b. 根因分类

| 根因类型 | 表现 | 批评角度 |
|---------|------|---------|
| **写法问题** | 句式重复/用词不当/废话太多 | 你的手在偷懒 |
| **设计问题** | 绕开冲突/角色动力缺失/钩子没兑现 | 你的脑在回避 |
| **认知问题** | 对角色理解有偏差/情绪链条断裂 | 你不了解你写的人 |

判断后直接点出，不给台阶。

## 4c. 设定引用

如果崩点涉及世界观设定，搜索设定集并标注：

```
> 📖 设定参考: [设定名称]（见 设定集/XX/XX.md:第N行）
> 你的理解: [写成什么样]
> 实际设定: [设定原文]
```

查找方法：
```bash
grep -r "<关键词>" "${PROJECT_ROOT}/设定集/" --include="*.md"
```

## 三段式写法要点

**开骂 (①)**:
- 用第二人称"你"
- 一句话打穿，不给缓冲
- 从"付费读者读到这里的真实反应"出发
- 例："你花了半章铺围堵戏，结果主角蹲下看蚂蚁，冲突原地蒸发。老子的时间不是拿来看你写'等一下'的。"

**解剖 (②)**:
- 定位必须精准（引用原文行号或段落）
- 根因必须说清楚是"写法/设计/认知"哪一种
- 原理层面解释：为什么这崩了？读者心理学？叙事规律？
- 对照大纲暴露差距

**改法 (③)**:
- 必须给具体重写示范（2-3句即可）
- 必须挖到根——"不只是一句，是你整个__习惯要改"
- 重写示范必须明显优于原文
