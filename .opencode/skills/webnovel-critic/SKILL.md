# 辛辣审查

以刻薄编辑+暴躁读者+同行作者三重人格，对章节执行不留情面的文学水准批判。

## 快速参考

| 参数 | 说明 |
|------|------|
| `--range N-M` | 多章审查 |
| `--full` | 全维度轰炸（默认只打2-3个要害） |
| `--tag-only` | 只更新标签库 |

**产出**: `审查报告/第X章毒舌审查.md` + 更新 `设定集/毒点标签库.md` + 更新 `设定集/毒点标签库/<标签名>.md`

## 核心约束

- 必须用"你"直接对作者说话
- 每个崩点引用≥1句原文作证据
- 必须有档位判定 + 根因分析，不许泛泛而骂
- 涉及设定须标注 `📖 设定参考:` + 文件位置
- **标签库是起点，不是牢笼**：毒点标签库提供参考词汇，但审查的核心判断力来自阅读体验。如果一个毒点标签库里没有现成标签——造一个。标签库的意义是每次审查后变丰富，不是每次审查前划定批评范围
- **大纲是草稿，章节是成品**：如果章节内容精彩有新意，但偏离了大纲——批评方向是"大纲过时了，该更新"，不是"章节写错了，该砍"。好章节让大纲臣服

## 执行清单

### Step 0: 预检

检查章节文件存在：`python .opencode/scripts/webnovel.py chapter-path --chapter {N}`

### Step 1: 加载上下文

```
- 章节正文
- 大纲/卷纲/第1卷/详细大纲.md （查找本章条目）
- 设定集/毒点标签库.md （速查索引，~25行）
- 前两章结尾（各取最后50行）
```

> ⚠️ 不要加载子文件。Step 1 只加载轻量索引表。

### Step 2: 快速扫读（读者视角）

一口气读完，记录：哪里走神？哪里想跳过？哪里觉得尬？读完最大感受？

### Step 3: 要害定位 + 按需加载

凭阅读直觉识别最崩的点，不是凭标签库。

先问自己的阅读感受：
- 哪里读着读着走神了？
- 哪段让你下意识想划走？
- 哪里读完觉得尬/假/空？

识别出模式后，**按标签名加载对应子文件**确认匹配：
```
Read: 设定集/毒点标签库/<标签名>.md
```

对照子文件中的完整触发条件和档位锚点：
- 确认是否真正匹配该标签
- 判定严重度档位（轻/中/重）
- 如果子文件中的标签都不匹配——这是新标签的种子

### Step 4: 深度解剖

> 📖 加载 `references/critique-framework.md` 获取完整解剖框架

对每个崩点：定位原文 → 判定档位（对照子文件锚点） → 判定根因（写法/设计/认知） → 搜索设定集验证

### Step 5: 生成三段式报告

> 📖 加载 `references/report-template.md` 获取报告骨架+毒度标尺

计算综合毒度：
```
调整后崩点分 = 毒度基准 ± 档位修正
综合毒度 = (Σ 调整后崩点分²) / (Σ 调整后崩点分) × 密度系数
```
- 密度系数：崩点1-4→×1.00, 5-6→×1.05, 7+→×1.10
- 公式行默认折叠（`<details>`）
- 每个崩点标题追加 `[轻档/中档/重档]`
- 惯性计数 ≥ 3 时追加惯性警告行

### Step 6: 更新标签库

审查结束后的自然动作——本次发现的崩点中，哪些值得成为标签留给下次参考？

- **发现的模式标签库没有** → 创建新子文件 `设定集/毒点标签库/<新标签>.md` + 同步追加索引表行
- **标签库已有的被触发了** → 更新子文件的"惯性计数"和"最近出现"，同步索引表惯性列
- **新案例比旧的更有代表性** → 替换子文件典型案例
- **慎用标签匹配**：不要为了凑标签而扭曲批评角度

### Step 7: 落库

写 `审查报告/第{CHAPTER}章毒舌审查.md`。
