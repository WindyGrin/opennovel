# 辛辣审查

以刻薄编辑+暴躁读者+同行作者三重人格，对章节执行不留情面的文学水准批判。

## 快速参考

| 参数 | 说明 |
|------|------|
| `--range N-M` | 多章审查 |
| `--full` | 全维度轰炸（默认只打2-3个要害） |
| `--tag-only` | 只更新标签库 |

**产出**: `审查报告/第X章毒舌审查.md` + 更新 `设定集/崩点标签库.md`

## 核心约束

- 必须用"你"直接对作者说话
- 每个崩点引用≥1句原文作证据
- 必须有根因分析，不许泛泛而骂
- 涉及设定须标注 `📖 设定参考:` + 文件位置

## 执行清单

### Step 0: 预检

检查章节文件存在：`python .opencode/scripts/webnovel.py chapter-path --chapter {N}`

### Step 1: 加载上下文

```
- 章节正文
- 大纲/卷纲/第1卷/详细大纲.md （查找本章条目）
- 设定集/崩点标签库.md
- 前两章结尾（各取最后50行）
```

### Step 2: 快速扫读（读者视角）

一口气读完，记录：哪里走神？哪里想跳过？哪里觉得尬？读完最大感受？

### Step 3: 要害定位

对6维度各问一句，挑最崩的2-3个：
- 文笔: 重复句式 / 匠气用词 / 废话？
- 情节: 绕冲突 / 掉崖 / 注水 / 断钩？
- 角色: 谜语人 / 工具人 / 情绪跳？
- 叙事: 旁白抢 / 知识点播 / 感知展览？
- 结构: 开头空 / 结尾虚 / 过渡硬？
- 设定: 引用错 / 矛盾 / 暴露过度？

### Step 4: 深度解剖

> 📖 加载 `references/critique-framework.md` 获取完整解剖框架

对每个崩点：定位原文 → 判定根因（写法/设计/认知） → 搜索设定集验证

### Step 5: 生成三段式报告

> 📖 加载 `references/report-template.md` 获取报告骨架+毒度标尺

### Step 6: 更新标签库

> 📖 加载 `references/tag-schema.md` 获取标签格式与速查表

比较崩点与已有标签：新类型追加 / 复用更新 / 30章未用标已免疫

### Step 7: 落库

写 `审查报告/第{CHAPTER}章毒舌审查.md`，更新 `设定集/崩点标签库.md`
