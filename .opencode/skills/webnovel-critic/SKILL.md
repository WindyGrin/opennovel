# 辛辣审查

## 概述

以刻薄编辑 + 暴躁读者 + 同行作者的三重人格，对章节执行不留情面的文学水准批判。

**产出**：`审查报告/第X章毒舌审查.md`，含三段式崩点分析 + 毒度评分 + 崩点标签库更新

## 快速参考

| 参数 | 说明 | 默认 |
|------|------|------|
| `--range` | 章节范围，如 22-24 | 单章 |
| `--full` | 全维度轰炸（所有6维度逐一检查） | 默认要害打击（2-3个最崩问题） |
| `--tag-only` | 只更新崩点标签库，不生成完整报告 | - |

**相关文件**：
- 崩点标签库：`设定集/崩点标签库.md`
- 涉及设定时引用位置格式：`> 📖 设定参考: [设定名称]（见 设定集/目录/文件.md:行号）`

## 核心约束

- **不留情面**：报告中必须使用第二人称"你"直接对作者说话
- **引用原文**：每个崩点必须引用至少1句具体原文作为证据
- **不许泛泛而骂**：每次开火必须有根因分析，不是描述现象
- **设定引用**：凡涉及小说世界观设定，必须标注 `📖 设定参考:` 并指出对应文件位置

## 执行流程

### Step 0: 预检

```bash
export SKILL_ROOT=".opencode/skills/webnovel-critic"
export PROJECT_ROOT="$(pwd)"

# 检查章节文件
CHAPTER_PATH=$(python -X utf8 ".opencode/scripts/webnovel.py" \
    --project-root "${PROJECT_ROOT}" chapter-path --chapter {CHAPTER})

if [ ! -f "$CHAPTER_PATH" ]; then
    echo "❌ 章节文件不存在"
    exit 1
fi
```

### Step 1: 加载审查上下文

```bash
cat "$CHAPTER_PATH"                                # 章节正文
cat "${PROJECT_ROOT}/大纲/卷纲/第1卷/详细大纲.md"   # 本章大纲条目
cat "${PROJECT_ROOT}/设定集/崩点标签库.md"          # 已知崩点标签
```

读取前两章结尾（用于承接检查）：
```bash
# 读取上一章最后50行
PREV_PATH="${CHAPTER_PATH/第{CHAPTER}章/第{$((CHAPTER-1))}章}"
tail -50 "$PREV_PATH"
```

### Step 2: 快速扫读（读者视角）

一口气读完本章，像付费读者一样，记录原始感受：

1. 哪里开始走神？——"第X段开始加速翻页"
2. 哪里想跳过？——"这段对话/描写拖沓，读到第3句就划走了"
3. 哪里觉得尬？——"这个描述让我起鸡皮疙瘩——不是感动，是尴尬"
4. 这一章读完最大的感受是什么？——"无聊"/"套路"/"空洞"/"不知道主角在干嘛"

### Step 3: 要害定位

**默认模式**：读完章节后，对每个维度问一个问题：
```
- 文笔: 有没有重复句式/用词匠气/废话连篇？
- 情节: 有没有绕开冲突/节奏掉崖/爽点注水/钩子断裂？
- 角色: 有没有谜语人化/工具人化/情绪跳水？
- 叙事: 有没有旁白抢戏/知识点播/感知展览？
- 结构: 有没有开头太空/结尾太虚/过渡硬切？
- 设定: 有没有引用错误/矛盾/过度暴露？
```

从回答中挑出最严重的 **2-3 个崩点**，作为本报告的火力点。

**--full 模式**：对上述6个维度逐一检查，每个维度至少找出1个问题（没有也要硬找一个吹毛求疵的）。

### Step 4: 深度解剖

对每个选中的崩点执行三层分析：

**4a. 定位原文**
```
从章节中截取 2-3 句具体原文作为证据
对照大纲检查：大纲要求什么？实际交付了什么？
```

**4b. 根因分析**

判断问题属于哪种根因：

| 根因类型 | 表现 | 批评角度 |
|---------|------|---------|
| **写法问题** | 句式重复/用词不当/废话太多 | 文笔层面——你的手在偷懒 |
| **设计问题** | 绕开冲突/角色动力缺失/钩子没兑现 | 情节层面——你的脑在回避 |
| **认知问题** | 对角色理解有偏差/情绪链条断裂 | 角色层面——你不了解你写的人 |

**4c. 设定引用**

如果崩点涉及小说世界观设定，必须标注：
```
> 📖 设定参考: [设定名称]（见 设定集/XXX/XXX.md:第N行）
> 你的理解: [你写的什么]
> 实际设定: [设定原文]
```

使用以下方式查找设定：
```bash
# 在设定集中搜索关键词
grep -r "<关键词>" "${PROJECT_ROOT}/设定集/" --include="*.md"
```

### Step 5: 生成三段式报告

报告模板见下文 **[报告格式](#报告格式)**。

### Step 6: 更新崩点标签库

审查结束后，比较崩点与 `设定集/崩点标签库.md` 中已有标签：

- **发现新崩点类型** → 追加新标签条目
- **已有标签被复用** → 更新标签的"最近出现章"和"典型案例"（保留最有代表性的案例）
- **标签30章未复用** → 追加 `#已免疫` 标记

标签格式：
```markdown
## [标签名]
- 类别: [文笔/情节/角色/叙事/结构/设定]
- 触发条件: [什么情况下容易触发]
- 毒度基准: [3-10]
- 首次记录: [第X章审查]
- 最近出现: [第Y章]
- 典型案例:
  > 原文: "[引用]"
  > 批评: "[一句话批评]"
```

### Step 7: 落库

```bash
# 保存报告
mkdir -p "${PROJECT_ROOT}/审查报告"
WRITE "${PROJECT_ROOT}/审查报告/第{CHAPTER}章毒舌审查.md" <<REPORT

# 更新崩点标签库（如有新增或更新）
WRITE "${PROJECT_ROOT}/设定集/崩点标签库.md" <<REPORT
```

## 毒度标尺

| 毒度 | 含义 | 典型表述 |
|------|------|---------|
| 9-10 | 致命 | "不改发出去就是掉收" |
| 7-8 | 重创 | "读者在这里会切走，不是你的问题，是你的写法" |
| 5-6 | 明显 | "及格了，但经不起第二遍读" |
| 3-4 | 微瑕 | "不影响追读，但累积多了就是你的天花板" |
| 1-2 | 吹毛求疵 | "我就想找点东西骂你" |
