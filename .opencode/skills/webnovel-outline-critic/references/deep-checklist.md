# Deep模式：4项跨文档对照清单

> 在quick的6维之外追加。需要加载全卷纲+核心设定+正文抽样。每个维度1-3个崩点。

## D1：总纲↔卷纲一致性

**对照源**：总纲的(反派分层/核心冲突/主角成长线) ↔ 全卷纲的详细大纲

**验证逻辑**：
1. 总纲"反派分层"表中每卷的反派——在对应卷的详细大纲中是否有明确出场条目？
2. 总纲"主角成长线"的8个关键跃迁节点——在卷纲中是否有对应章号？
3. 总纲"卷划分"表中每卷的核心冲突——卷纲的节拍表是否围绕该冲突展开？
4. 总纲"世界观/力量体系"中定义的机制——在卷纲中是否有对应的情节使用？

**常见崩点**：反派承诺未兑现、成长节点空洞、核心冲突被替换

## D2：伏笔表闭环

**对照源**：总纲(伏笔表) ↔ 全卷详细大纲

**验证逻辑**：
1. 逐条检查伏笔表，"埋设章"在对应卷的详细大纲中是否有条目？
2. "回收章"在对应卷的详细大纲中是否有条目？
3. 存在孤儿伏笔（只埋不收——有埋设章但回收章对应的卷纲不存在）？
4. 存在幽灵伏笔（只收不埋——有回收章对应的卷纲条目但埋设章无对应）？
5. 核心伏笔（标记为"核心"的伏笔）埋设和回收之间是否有足够的展开空间？

**常见崩点**：伏笔孤儿、埋收距离过近、核心伏笔被降级

## D3：卷间节拍密度

**对照源**：全卷节拍表 + 全卷Strand分布 + 爽点规划

**验证逻辑**：
1. 逐卷计算Quest/Fire/Constellation占比——
   - Quest应该占主导（55-65%）
   - Fire应该在20-30%
   - Constellation应该控制在10-20%
   - 是否有某卷某Strand占比超过上限2倍？
2. 爽点规划中每卷的"卷末大爽点"——是否在卷纲详细大纲中有对应的高潮章？
3. 卷间爽点密度是否均匀？是否存在某卷爽点过于稀疏？
4. Fire线（情感/关系）是否每卷都有至少一条发展线？

**常见崩点**：Strand严重失衡、卷末大爽点空洞、Fire线干旱

## D4：大纲↔正文兑现率

**对照源**：详细大纲(核心爽点/冲突) ↔ 正文抽样(卷首/卷中/卷末各2章)

**验证逻辑**：
1. 大纲规划的卷首章的核心爽点——在正文中是否实际写出？
2. 大纲规划的卷中关键冲突——在正文中是否实际发生？
3. 大纲规划的卷末高潮——在正文中是否兑现？
4. 是否存在"大纲写了但正文根本没提"的核心事件？

**常见崩点**：大纲架空（大纲写得热闹，正文影子都没有）、兑现将就（大纲规划的大爽点在正文中弱化为小爽点）

**正文抽样策略**：
- 只抽样卷首2章、卷中2章、卷末2章（不需要全卷加载）
- 每章只读结尾50行（不需要全文）
- 目的不是全面审查正文质量——是验证大纲有没有被正文执行
