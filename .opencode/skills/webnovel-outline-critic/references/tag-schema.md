# 大纲缺陷标签格式与速查

## 文件架构

索引表位于 `设定集/大纲缺陷标签库.md`，子文件位于 `设定集/大纲缺陷标签库/<标签名>.md`。

审查时先扫索引表识别模式，按需加载匹配的子文件确认档位锚点。

## 子文件条目格式

```markdown
# 标签名

- 类别: [结构/逻辑/节奏/一致性/规划]
- 触发条件: [完整触发条件]
- 毒度基准: [3-10]
- 档位锚点:
  | 轻(X) | [轻档判断标准] |
  | 中(Y) | [中档判断标准] |
  | 重(Z) | [重档判断标准] |
- 首次记录: [第X章或日期]
- 最近出现: [审查目标]
- 惯性计数: [N]

## 典型案例
> 原文: "[大纲条目引用]"
> 批评: "[一句话]"
```

## 档位说明

| 档位 | 修正 | 含义 |
|------|:---:|------|
| 轻 | 基线 - 2 | 刚触及触发线，影响局部 |
| 中 | 基线 ± 0 | 典型触发，符合标签定义 |
| 重 | 基线 + 2 | 远超触发线，影响整卷/全篇结构 |

调整后分值钳制 [1, 10]。

## 更新规则

| 情况 | 操作 |
|------|------|
| 发现新崩点类型 | 创建新子文件 + 同步追加索引表行 |
| 已有标签被复用 | 更新子文件的"最近出现"和惯性计数，同步索引表惯性列 |
| 标签30次审查未复用 | 追加 `#已免疫` 标记 |
| 新案例更优 | 替换子文件的典型案例 |
