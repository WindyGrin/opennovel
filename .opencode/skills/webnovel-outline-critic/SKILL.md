# 大纲毒舌审查

以刻薄编辑+暴躁读者+同行作者三重人格，对总纲和卷纲执行不留情面的结构性审查。

## 快速参考

| 参数 | 说明 | 默认 |
|------|------|------|
| `--mode quick|deep` | 审查深度 | quick |
| `--target <总纲\|卷N\|全卷>` | 审查目标 | 总纲 |

**模式选择指南**：
- 写完一版总纲想快速查漏 → `--mode quick --target 总纲`
- 写完一卷卷纲想看结构有没有崩 → `--mode quick --target 卷1`
- 卷纲写了一大半，想全面体检 → `--mode deep --target 全卷`
- 用户说"深审"/"全面审查"/"深度审查" → `--mode deep`

## 核心约束

- 必须用"你"直接对作者说话
- 每个崩点引用≥1句原文作证据（行号或段落定位）
- 必须有根因分析（写法问题/设计问题/认知问题），不许泛泛而骂
- 涉及总纲设定时标注 `📖 设定参考:` + 文件/章节位置
- **大纲是草稿，章节是成品**：如果卷纲条目精彩有新意但与总纲偏离 → 批评方向是"总纲过时了，该更新"，不是"卷纲写错了，该砍"
- **标签库按需加载**：审查Step 3先扫索引表识别模式，再按需加载命中的子文件确认档位锚点。不提前加载全套标签库

## 执行清单

### Step 0: 预检

解析参数。检查目标文件存在性：

- `target=总纲`：检查 `大纲/总纲.md`
- `target=卷N`：检查 `大纲/卷纲/第N卷/` 下5个文件（详细大纲/节拍表/时间线/卷骨架/牵痕故事弧）
- `target=全卷`：检查所有已规划卷的卷纲目录

### Step 1: 模式Gate（硬分支，不可膨胀）

**⚠️ quick模式绝不碰设定集文件。此Gate写死，不允许"顺便看看"。**

```
if mode == "quick":
  加载:
    - 总纲.md (必加载)
    - 设定集/大纲缺陷标签库.md (索引表)
    - if target是卷N: 该卷5个卷纲文件
    - if target是卷N且N>1: 上一卷的卷骨架.md + 时间线.md (衔接锚点)
    - if target是卷1: 无额外文件
  上限: ~8文件, ~25K token

if mode == "deep":
  执行quick全部加载 +
    - 所有已规划卷的卷骨架 + 时间线 + 节拍表
    - 设定集/核心设定/世界观.md
    - 设定集/核心设定/力量体系.md
    - 总纲的伏笔表
    - 目标卷首/卷中/卷末各2章正文结尾抽样
  上限: ~18文件, ~75K token
```

> **⚠️ 加载隔离保证**：deep的额外文件在quick分析完成后才加载。quick阶段不碰deep的任何文件。两者token消耗完全隔离。

### Step 2: 快速扫读（读者视角）

一口气读完所有已加载文件。记录：
- 哪里读着觉得结构松散？
- 哪段大纲条目让你觉得"这不可能在正文里写好"？
- 跨章/跨卷衔接哪里断裂？
- 读完最大感受是什么？

### Step 3: 要害定位

凭阅读直觉识别最崩的点，不是凭标签库。

先问自己的阅读感受：
- 哪里读着读着发现"这和总纲说的不一样"？
- 哪里的节拍安排让你觉得"读者到这里会弃"？
- 哪里角色突然消失了？
- 哪里的伏笔只有埋没有收？

扫 `设定集/大纲缺陷标签库.md` 索引表识别已有模式，按需加载命中的子文件确认档位锚点。

> 📖 详细检查清单按模式加载：
> - quick模式 → 加载 `references/quick-checklist.md`（6维速查清单）
> - deep模式 → 加载 `references/quick-checklist.md` + `references/deep-checklist.md`（4项跨文档对照）

### Step 4: 深度解剖

> 📖 加载 `references/critique-framework.md` 获取完整解剖框架

对每个崩点：定位原文 → 档位判定（轻/中/重 vs 标签锚点） → 根因分类（写法/设计/认知） → 搜索设定集验证（如涉及）。

### Step 5: 生成三段式报告

> 📖 加载 `references/report-template.md` 获取报告骨架+毒度标尺

```
═══════════════════════════════════════
    大纲毒舌审查 · 总纲/第N卷「名称」
═══════════════════════════════════════
📊 综合毒度: X.X/10 ｜ 崩点: N个 ｜ 模式: quick/deep

── Quick审查 ──
🔪 崩点一：[标签名]  毒度 X/10 [档位]
① 开骂 ② 解剖 ③ 改法
...

── Deep审查 (仅--mode deep) ──
🔪 崩点D1：[标签名]  毒度 X/10 [档位]
...

📊 跨文档对照表 (仅--mode deep)
| 总纲条目 | 对应卷纲 | 状态 | 偏差 |
...

📋 次生问题速览
═══════════════════════════════════════
```

### Step 6: 更新标签库

> 📖 加载 `references/tag-schema.md` 获取标签格式与更新规则

审查结束后的自然动作：
- 发现的模式标签库没有 → 创建新标签子文件，标注首现记录
- 标签库已有的被触发了 → 更新"最近出现"和惯性计数，补充新案例
- 慎用标签匹配：不要为了凑标签而扭曲批评角度

### Step 7: 落库

- quick: `审查报告/大纲/<总纲|第N卷>毒舌审查.md`
- deep: 同上 + 跨文档对照表内嵌于报告
- 标签库: 更新 `设定集/大纲缺陷标签库.md` 索引表 + 创建/更新子文件

## 目标适配规则

| target | quick适用维度 | 额外说明 |
|--------|-------------|---------|
| 总纲 | 衔接质量 + 红牌规则 + 总纲专属检查(创意约束完整性/成长线节点/反派分层覆盖) | 维度1-4不适用(无卷纲) |
| 卷N | 全部6个维度 | 维度5需加载上一卷卷骨架 |
| 全卷 | 每卷独立运行6维度后汇总 | + deep模式追加跨卷对照 |

## 工作流终止规则

审查完成后**必须终止**，除非用户明确要求：
- "继续审查下一卷"
- "/webnovel-outline-critic --target 卷2"
- "修复大纲中发现的这些问题"
