<context>
此文件用于章节质量快速自检。Claude 已知一般写作错误，这里只列举网文创作特定的常见问题。
</context>

<instructions>

## 快速自检清单

| 检查项 | 通过标准 | 违规后果 |
|--------|----------|----------|
| 战力合理 | 主角实力 ≤ state.json | 破坏战力体系 |
| 爽点存在 | 至少 1 个爽点/章 | 读者流失 |
| 实体识别 | 新实体可被 Data Agent 自动提取 | 设定混乱 |
| 人物一致 | 对话/行为符合人设 | OOC 出戏 |
| 节奏平衡 | 未连续 5+ 章同一 Strand | 审美疲劳 |
| 伏笔追踪 | 伏笔可被识别并记录 | 遗忘伏笔 |

</instructions>

<examples>

<example>
<input>错误：主角筑基 3 层，打败金丹期敌人</input>
<output>
❌ **战力崩坏**

✅ 修正：如必须越级战斗，需要：
- 金手指/秘法加持
- 对手受伤/被削弱
- 环境优势/陷阱辅助
</output>
</example>

<example>
<input>错误：整章主角赶路，无冲突无爽点</input>
<output>
❌ **爽点缺失**

✅ 修正：赶路途中安排：
- 遭遇事件（强盗/妖兽/奇遇）
- 巧遇角色（伙伴/敌人/女主）
- 发现隐藏地点/宝物
</output>
</example>

<example>
<input>错误：突然出现"紫霄宗"，设定集中无此势力</input>
<output>
❌ **擅自发明**

✅ 修正：
- 确保新势力有明确名称和描写
- Data Agent 会自动提取并询问是否加入设定集
</output>
</example>

<example>
<input>错误：主角性格"隐忍冷静"，突然暴怒失控</input>
<output>
❌ **人物 OOC**

✅ 修正：
- 参考角色卡性格
- 如需转变须有铺垫（触及底线/重大刺激）
- 标注性格变化原因
</output>
</example>

<example>
<input>错误：反派台词全是"你这个蝼蚁！""不自量力！"</input>
<output>
❌ **对话脸谱化**

✅ 修正：
- C级反派可脸谱化
- B级以上应有智商和个性
- 嘲讽要自然且有信息量
</output>
</example>

<example>
<input>错误：一段内连用3个"不是...而是..."句式
"沈安看的不是蚂蚁。是蚂蚁脚下的道痕。那道气机不是自然形成的。是人。不是突破。是有人在石头上刻了一刀。"
</input>
<output>
❌ **句法重复（"不是...而是..."滥用）**

✅ 改写为独立陈述+因果衔接：
"沈安看的不是蚂蚁——蚂蚁只是他追踪道痕的路径。那道气机有割裂感，点状，收放极快。有人在石头上用道痕刻了一刀，刚刻完，立刻收敛。突破是扩散的，这道是精准的暗杀手法。"
——同一段内只保留1处"不是"作为强调，其余改为直接描写。
</output>
</example>

<example>
<input>错误：全书已确立的隐喻反复加引号
"之前奉命来"磨"他" "默认多做比"刚好"强"
</input>
<output>
❌ **隐喻加引号（匠气）**

✅ 去引号——读者已建立认知，直接使用更自然：
"之前奉命来磨他" "默认多做比刚好强"
只在首次引入或特指术语时保留引号（如宗规原文引用）。
</output>
</example>

<example>
<input>错误：主角全章行为模式="察觉异常→说一句谜语→等人追问→不解释→留白"。每章至少一处"只有我能看到"的感知展示。
"他感知到后山方向有一道气机今天早上没有复位。不是筛子的——是另一道。这一步不是犹豫——是他感知到..."
</input>
<output>
❌ **谜语人表演感（大佬逼格衰减）**

修正三原则：
1. 能说完的话说完："后山今天有人来过。"
2. 让读者推断，不让旁白辩护：去掉"这一步不是犹豫——是他感知到..."
3. 减少感知展示频率：不是每章都需要新发现。允许主角有一天什么都不注意到。
4. 真正的大佬不屑于当谜语人——他想说就说，不想说就走，不需要靠半句话制造悬念。
</output>
</example>

<example>
<input>错误：叙事站在主角角度对某个现象进行长篇理论分析，读起来像教科书。
"苍渊的行军蚁种群有一种习惯：它们沿着地脉道痕迁徙。不是蚂蚁懂得道痕理论——是它们的身体比重和地面道痕的微弱磁性刚好形成了一个无意识的跟随路线..."
</input>
<output>
❌ **理论灌输式叙事（破坏沉浸感）**

修正：压缩为一两句行动描写，让读者自行脑补原理。
"地上有一只蚂蚁，正沿着那根细线往前爬，走到拐弯的地方时忽然改了方向——绕了一个小圈，从另一侧兜过去了。"
叙事不需要对每个现象做学术阐述。
</output>
</example>

</examples>

<errors>
❌ 战力崩坏（越级秒杀无理由） → ✅ 检查 state.json，合理设定对手
❌ 爽点缺失（整章无冲突） → ✅ 至少 1 个爽点，过渡章也要有
❌ 擅自发明（新实体描写模糊） → ✅ 确保新实体有明确名称和描写
❌ 人物 OOC（性格突变无铺垫） → ✅ 参考角色卡，有转变需铺垫
❌ 节奏单调（连续 10 章纯战斗） → ✅ 检查 strand_tracker，切换 Strand
❌ 伏笔遗忘（超 100 章未回收） → ✅ 核心伏笔 50 章内回收
❌ 对话脸谱化（反派只会喊口号） → ✅ B级以上反派要有智商
❌ "不是...而是..."句式泛滥（每段一个，句法重复） → ✅ 同一章内不超过3次，超过则改写为独立陈述或因果句式；通过换个主语、换个语序、直接描写来替代
❌ 核心隐喻加引号（如"磨""及格"等全书已确立的概念反复加引号） → ✅ 这些隐喻在读者认知中已建立，无需双引号标注，直接使用降低匠气
❌ 叙事节奏单一（所有段落"不是X——是Y"结构，无句法变化） → ✅ 每段检查句法结构多样性：至少保留1-2种不同句式（因果句/并列句/疑问句/直接描写句）
❌ 谜语人表演感（每章必有"只有主角能感知到"的新发现，说半句留半句等人追问） → ✅ 允许主角有不注意到任何事情的章；能说完的话说完；大佬不屑当谜语人
❌ 理论灌输式叙事（对现象进行长篇幅学术阐述，破坏小说沉浸感） → ✅ 压缩为行动描写，让读者自行脑补原理；旁白不需要替主角辩护
</errors>
