# 毒度基准打分体系重构设计

> 状态: 待实施
> 日期: 2026-05-12
> 关联: 设定集/毒点标签库.md, .opencode/skills/webnovel-critic/

---

## 1. 问题诊断

当前毒度体系存在五个核心缺陷：

| 问题 | 表现 | 影响 |
|------|------|------|
| 静态基线死板 | 23个标签只有5个基准值(5/6/7/8/9)，同标签无论严重程度如何都是同一分值 | 无法区分"句式复读机出现5次"和"出现20次" |
| 聚合算法黑箱 | 综合毒度公式从未文档化，审查器自由裁量 | 跨章不可比，同组崩点不同章节可能得出不同综合分 |
| 层级太粗 | 五档标尺每档跨度2分，23个标签挤在5个数值上 | 标签间拉不开差距，低区分度 |
| 缺少动态维度 | 无频次、密度、跨章惯性的系统化追踪 | 同一标签连续5章触发和首次触发分值相同 |
| Token 浪费严重 | 每次审查加载全量321行标签库，含27条已废弃条目和冗余案例 | 标签库越大，审查成本越高，抑制标签生长 |

## 2. 设计原则

- **向后兼容**: 现有23个标签的毒度基准全部保留不动
- **增量改进**: 在现有流程上叠加修正层，不推翻审查工作流
- **透明可验**: 综合毒度的计算过程对作者可见（默认折叠）
- **经验优先**: 档位锚点是参考线，审查器的阅读体验判断仍为第一优先级
- **按需加载**: 审查时只加载速查索引，匹配标签后才加载完整条目，Token 消耗与标签库规模解耦

- 当前全量加载: ~8000 chars（321行）

### 3.1 严重度档位（Per-Instance Severity Tier）

每个崩点实例在判定时附加一个严重度档位，修正标签的基线分。

| 档位 | 含义 | 修正 | 判断标准 |
|------|------|------|---------|
| 轻 | 刚触及触发线，影响局部 | 基线 - 2 | 读者可能注意不到，或注意到但不影响阅读节奏 |
| 中 | 典型触发，标签定义的标准程度 | 基线 ± 0 | 符合标签典型案例的严重程度 |
| 重 | 远超触发线，影响整章阅读体验 | 基线 + 2 | 读者会因此加速翻页或产生弃读冲动 |

- 调整后分值钳制在 [1, 10] 范围
- 示例: 句式复读机(基线7) → 轻档=5, 中档=7, 重档=9

### 3.2 综合毒度公式

```
综合毒度 = (Σ 调整后崩点分²) / (Σ 调整后崩点分)
```

本质是**二次均值**——高分崩点的权重天然更重。一个毒度9的致命崩点 + 两个毒度5的微瑕，比三个毒度6的崩点综合分更高。

**密度修正系数**（崩点数 ≥ 5 时触发）:

| 崩点数 | 系数 |
|--------|------|
| 1-4 | ×1.00 |
| 5-6 | ×1.05 |
| 7+ | ×1.10 |

最终综合毒度 = 二次均值 × 密度系数，保留一位小数。

**边界情况**: 0崩点章节 → 不计算综合毒度，报告标注"本章无明显崩点"。

**惯性警告**不参与数值计算，在报告中独立展示为警告行。

### 3.3 标签库条目格式升级

每个标签新增「档位锚点」和「惯性计数」字段:

```markdown
## 句式复读机

- 类别: 文笔
- 触发条件: 同一章节内相同句式（如"不是X而是Y"）出现≥5次
- 毒度基准: 7
- 档位锚点:
  | 轻(5) | 5-7次，出现在非关键段落，不打断核心场景节奏 |
  | 中(7) | 8-12次，开始形成可感知的模式，影响阅读流畅度 |
  | 重(9) | 13次+，或核心场景中出现密集复用，读者产生烦躁感 |
- 首次记录: 第23章毒舌审查
- 最近出现: 第23章
- 惯性计数: 0
- 典型案例: ...
```

**字段说明**:
- `毒度基准`: 保持不变，代表"中档"的典型分值
- `档位锚点`: 三个档位的具体判断参考线（人工定义，经验校准）
- `惯性计数`: 连续触发章数，每次复用 +1，中断则归零。≥3 时触发惯性警告

**「大纲脱节」特殊处理**（保持不变）:
- 章节差 → 正常崩点，毒度8，档位按偏离严重度判定
- 章节好 → 不评定为崩点，标注「大纲需更新」，不计入综合毒度

### 3.4 审查报告输出升级

报告头部增加公式透明度和惯性警告，崩点卡片增加档位标注:

```
═══════════════════════════════════════
       毒舌审查 · 第X章「标题」
═══════════════════════════════════════

📊 综合毒度: 7.7/10  ｜  崩点: 3个  ｜  模式: 要害打击
   ▶ 公式: (8²+8²+7²)/(8+8+7) = 7.70  [默认折叠，点击展开]

⚠️ 惯性警告: 拐弯走冲突已连续第3章触发

───────────────────────────────────────
🔪 崩点一：拐弯走冲突  毒度 8/10 [重档]
───────────────────────────────────────
[开骂/解剖/改法 三段式不变]
───────────────────────────────────────
```

**变化点**:
- 公式行默认折叠（Markdown `<details>` 实现），需时展开
- 每个崩点标题追加 `[轻档/中档/重档]`
- 惯性警告独立一行，不混入数值
- 综合毒度保留一位小数

### 3.5 webnovel-critic 工作流变更

**审查流程按需加载改造**:

| 步骤 | 当前行为 | 改造后 |

**Skill 文件变更**:
- `SKILL.md`: Step 2 指令从"加载毒点标签库"改为"加载速查索引"，追加"匹配后加载对应子文件"步骤

## 6. 不做什么

- 不废除现有毒度基准值（向后兼容）
- 不为旧审查报告回溯重算（历史数据保持原样）
- 不把惯性警告自动换算为数值（避免过度叠加）
- 不引入章节位置趋势线（留待后续迭代，当前聚焦单章评分精度）
- 不为旧审查报告回溯匹配子文件（历史报告保持原引用格式）
